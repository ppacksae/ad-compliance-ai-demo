# 광고/배너 컴플라이언스 검토 지원 플랫폼 기획

---

## 프로젝트 개요

증권사 내부에서 광고/배너 소재를 제작·릴리즈하는 과정에서 발생하는  
컴플라이언스/법무 검토의 비효율을 AI Agent 기반 내부 플랫폼으로 해결하는 기획안입니다.

### 핵심 문제 정의

> **의사결정 컨텍스트의 휘발로 인한 제로베이스 반복 검토**

검토 근거와 판단 이력이 슬랙·메일·문서로 분산되어,  
매번 동일한 규정을 반복 설명하고 과거 선례를 수동 탐색하는 구조적 비효율이 존재합니다.

### 차별화 포인트

| # | 포인트 | 설명 |
|---|---|---|
| 01 | **설명 가능성 + Citation** | AI가 리스크 지적 시 정책 조항·선례를 명확히 인용하며 수정안 제시 |
| 02 | **OCR 기반 이미지 검토** | 배너 이미지 내 텍스트를 OCR로 추출해 금지어·규정 위반 탐지 |
| 03 | **Knowledge Flywheel** | 법무/컴플 판단 이력이 자동으로 고품질 선례로 축적되어 시간이 갈수록 정교해지는 구조 |

## 플랫폼 구조

### 두 가지 진입 경로

```
경로 A. AI 사전검토 (챗봇)
마케터가 기획 단계에서 문구/이미지를 자율 점검
→ 법무 공식 요청 전 셀프 교정

경로 B. 소재 제출 (공식 검토)
최종 배너 이미지 업로드 → OCR 기반 AI 선검토
→ 법무/컴플 최종 승인
```

### 화면 구성

| 역할 | 화면 | 설명 |
|---|---|---|
| MARKETER | AI 사전검토 | 챗봇 형태 선검토 · Citation 뷰어 |
| MARKETER | 소재 제출 | OCR 선검토 · AI 리포트 · 법무 요청 |
| MARKETER | 내 진행 현황 | 티켓별 처리 상태 추적 (My Board) |
| LEGAL & COMPLIANCE | 통합 심의 데스크 | 검토 대기 · 완료 이력 · 결재 진행 |
| LEGAL & COMPLIANCE | 처리 현황 | SLA · AI 성능 · Knowledge Flywheel 지표 |

### Agent 구성

```
Master Agent (오케스트레이터)
├── OCR Tool          이미지 텍스트 추출
├── Compliance Tool   금지어 · 고지 문구 체크  (병렬)
├── RAG Search Tool   3-Tier VDB 선례 검색     (순차)
└── Risk Scoring Tool HIGH / MEDIUM / LOW 판정
```

### 지식 베이스 3-Tier 구조

```
Tier 1. 정책/가이드 문서    금융규정 · 사내 심의기준 (기계적 적재)
Tier 2. 플랫폼 검토 결과    운영 후 자동 축적 · 가중치 높음
Tier 3. 법무 확정 선례      AI-법무 판단 불일치 케이스 · 최고품질
```

## 데모 실행 방법

```bash
npm install
npm run dev
```

브라우저에서 `http://localhost:5173` 접속

### 데모 시나리오

| 메뉴 | 시나리오 |
|---|---|
| AI 사전검토 | 예시 문구 카드 클릭 → AI 즉시 피드백 + Citation 뷰어 |
| 소재 제출 | 배너 시안 선택 → OCR 분석 → AI 리포트 확인 → 법무 요청 |
| 통합 심의 데스크 | 대기 소재 클릭 → AI 리포트 확인 → 승인/수정요청/반려 |
| 처리 현황 | Knowledge Flywheel 성장 추이 · SLA 준수율 확인 |

---

## 주요 설계 원칙

- **AI는 게이트키퍼가 아닌 보조자** — AI가 HIGH로 판정해도 마케터는 법무로 제출 가능
- **Human-in-the-loop** — 모든 최종 승인 권한과 법적 책임은 법무/컴플 담당자에게 귀속
- **감사 추적** — 모든 검토·수정·승인 이력이 플랫폼에 자동 기록
